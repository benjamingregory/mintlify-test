## Saving and Reusing Code with Coplay

One of Coplay's powerful features is the ability to save and reuse code snippets and patterns that you frequently need in your Unity projects.

## Common Unity Design Patterns

Coplay can help you implement standard design patterns optimized for Unity development. Here are some you can request:

### Singleton Pattern

Perfect for manager classes that should only have one instance:

```csharp
// Ask Coplay: "Create a singleton GameManager class"

public class GameManager : MonoBehaviour
{
    // Singleton instance
    public static GameManager Instance { get; private set; }

    private void Awake()
    {
        // Singleton implementation
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);
            return;
        }

        Instance = this;
        DontDestroyOnLoad(gameObject);
    }

    // Your game manager code here...
}
```

<Note>
  Coplay can customize any pattern to fit your specific project needs. Just
  provide context about your project when requesting a pattern.
</Note>

### Object Pooling

Efficient instantiation and reuse of frequently used objects:

```csharp
// Ask Coplay: "Write an object pooling system for bullets"

public class ObjectPool : MonoBehaviour
{
    [SerializeField] private GameObject prefab;
    [SerializeField] private int poolSize = 20;

    private List<GameObject> pool;

    private void Awake()
    {
        pool = new List<GameObject>(poolSize);
        InitializePool();
    }

    private void InitializePool()
    {
        for (int i = 0; i < poolSize; i++)
        {
            GameObject obj = Instantiate(prefab, transform);
            obj.SetActive(false);
            pool.Add(obj);
        }
    }

    public GameObject GetFromPool()
    {
        foreach (GameObject obj in pool)
        {
            if (!obj.activeInHierarchy)
            {
                obj.SetActive(true);
                return obj;
            }
        }

        // If all objects are in use, create a new one
        GameObject newObj = Instantiate(prefab, transform);
        pool.Add(newObj);
        return newObj;
    }

    public void ReturnToPool(GameObject obj)
    {
        obj.SetActive(false);
    }
}
```

<Warning>
  Requesting complex patterns may require multiple iterations with Coplay. Start
  with the basic pattern and then ask for specific customizations.
</Warning>

## Saving Your Conversation Templates

You can save your conversations with Coplay for future reference:

1. Have a productive conversation with Coplay about a specific Unity concept or implementation
2. Click the **Export** button in the Coplay interface
3. Choose a format to save the conversation:
   - **Full Conversation**: Includes all back-and-forth messages
   - **Code Only**: Extracts only the code snippets
   - **Summary**: A condensed version with key points

## Creating Script Templates

Coplay can generate template scripts for common Unity components:

### MonoBehaviour Template

```csharp
// Ask Coplay: "Create a template for a typical MonoBehaviour script"

using UnityEngine;

public class NewComponent : MonoBehaviour
{
    [Header("Configuration")]
    [SerializeField] private float someValue = 1.0f;
    [SerializeField] private GameObject someReference;

    // Private variables
    private Transform myTransform;

    #region Unity Lifecycle Methods

    private void Awake()
    {
        // Initialize references
        myTransform = transform;
    }

    private void Start()
    {
        // Initialize component
        Initialize();
    }

    private void Update()
    {
        // Per-frame logic
    }

    private void FixedUpdate()
    {
        // Physics-related updates
    }

    private void OnDestroy()
    {
        // Cleanup
    }

    #endregion

    #region Public Methods

    public void DoSomething()
    {
        // Implement functionality
    }

    #endregion

    #region Private Methods

    private void Initialize()
    {
        // Initialization logic
    }

    #endregion
}
```

## Using Coplay for Consistent Coding Style

You can ask Coplay to format your code according to your team's coding standards:

```
"Generate a C# script for player movement following our team's coding standards:
- Public fields with [SerializeField] instead of public variables
- Regions for organization
- Method names use PascalCase
- Private variables prefixed with underscore
- XML documentation for public methods"
```

## Saving Project-Specific Templates

For larger project-specific templates, you can:

1. Ask Coplay to generate a comprehensive template
2. Save it as a script in your Unity project's "Templates" folder
3. Reference it in future Coplay conversations:

```
"Please help me create a new enemy behavior following our project template in Templates/EnemyBehavior.cs, but modify it to be a ranged attacker"
```
